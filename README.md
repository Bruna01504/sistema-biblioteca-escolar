# 📚 BiblioEscola — Sistema de Gestão de Biblioteca

MVP desenvolvido para a disciplina de Análise e Projeto de Sistemas do 3º Ano do Ensino Médio Técnico (2º Trimestre de 2026).

## 🚀 Objetivo do Projeto
Transformar o controle físico de uma biblioteca escolar em um sistema digital ágil. O sistema permite a navegação por abas e exibe relatórios dinâmicos e filtrados por **Turno, Turma e Série** diretamente ao clicar nos cards informativos da tela inicial, otimizando a identificação de devoluções em atraso.

## 🛠️ Tecnologias Utilizadas
* **HTML5:** Estruturação semântica e mecanismo de divisão por abas.
* **CSS3:** Design moderno, responsivo (Flexbox/Grid) com a paleta de cores institucional (tons de roxo, lilás e branco).
* **JavaScript (ES6):** Manipulação de Arrays em memória para simulação de banco de dados, filtros avançados (`.filter()` e `.sort()`) e injeção dinâmica de tabelas via DOM (`.innerHTML`).

## 📋 Requisitos do Sistema

### Requisitos Funcionais (RF)
* **RF001:** O sistema deve permitir o cadastro de alunos contendo Nome, Turma e Turno.
* **RF002:** O sistema deve permitir o cadastro de livros (Título, Autor e Quantidade).
* **RF003:** O sistema deve exibir relatórios detalhados ao clicar nos cards de estatísticas.
* **RF004:** O relatório de Alunos e de Atrasos deve vir organizado e separado por Turno e Turma.

### Requisitos Não Funcionais (RNF)
* **RNF001:** Os dados devem ser processados e mantidos em memória (Arrays) nesta fase de MVP.
* **RNF002:** A interface deve ser limpa, intuitiva e usar transições visuais nos elementos clicáveis.
